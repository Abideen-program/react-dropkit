Why react-dropkit

Everything you need.
Nothing you don't.

Built to solve the real gaps in existing file upload libraries — without the bloat.

🎯

TypeScript-first

Full types out of the box. No @types/ package needed. Every prop, hook return, and callback is fully typed.

♿

Accessible by default

ARIA roles, keyboard navigation (Enter/Space), screen reader announcements, and focus management — baked in from day one.

⚡

ESM + CJS dual build

Works in Next.js App Router, Pages Router, Vite, Remix, and Create React App. Fixes the ESM breakage found in react-dropzone.

🖼️

Built-in previews

Images get real thumbnails generated from the file itself. Other file types show clean icons with extension badges. No upload required.

📊

Per-file progress

Each file gets its own animated progress bar. Status updates from Ready → Uploading → Uploaded or Failed with live ARIA announcements.

🧩

Headless or styled

Drop in <Dropzone /> for instant UI, or use useFileUpload() to build a completely custom experience.

🔒

Smart validation

Accepts MIME wildcards (image/*), exact types, and extensions. Cross-browser including the Windows/Safari MIME bug fix.

🔌

Zero backend opinions

Plug in your own upload logic — S3, Cloudinary, Supabase, your own API. The library handles UI state; you handle the transport.

🌲

Tree-shakeable

Import only what you use. If you only import useFileUpload, only that ships to your users.

BasicUpload.tsx 
import { Dropzone } from 'react-dropkit'

// Drop in and go. Drag-and-drop, previews, validation — all included.
export function BasicUpload() {
  return (
    <Dropzone
      multiple
      validation={{
        accept: ['image/*', '.pdf', '.docx'],
        maxSize: 10 * 1024 * 1024, // 10MB
        maxFiles: 5,
      }}
      onFilesAdded={(files) => console.log('Added:', files)}
      onFilesChange={(files) => console.log('Current files:', files.length)}
      onError={(err) => console.error(err)}
    />
  )
}
CustomUpload.tsx 
import { useFileUpload, FileList } from 'react-dropkit'

// Full control over every pixel. The hook handles all the logic.
export function CustomUpload() {
  const {
    files, isDragActive, isDragReject,
    getRootProps, getInputProps, inputRef,
    removeFile, uploadAll, clearFiles,
  } = useFileUpload({
    multiple: true,
    validation: { accept: ['image/*'], maxSize: 5 * 1024 * 1024 },
    onUpload: async (file) => {
      // your upload logic here
    },
  })

  return (
    <div>
      <div
        {...getRootProps()}
        style={{ border: `2px dashed ${isDragReject ? 'red' : isDragActive ? 'blue' : 'gray'}` }}
      >
        <input {...getInputProps()} ref={inputRef} />
        <p>{isDragActive ? 'Drop here!' : 'Click or drag files'}</p>
      </div>

      <FileList files={files} onRemove={removeFile} />

      {files.length > 0 && (
        <div>
          <button onClick={uploadAll}>Upload All</button>
          <button onClick={clearFiles}>Clear</button>
        </div>
      )}
    </div>
  )
}
ProfileForm.tsx 
import { useState } from 'react'
import { useFileUpload } from 'react-dropkit'

export function ProfileForm() {
  const [name, setName] = useState('')
  const { files, getRootProps, getInputProps, inputRef } = useFileUpload({
    multiple: false,
    validation: { accept: ['image/*'] },
  })

  const handleSubmit = async (e) => {
    e.preventDefault()
    const formData = new FormData()
    formData.append('name', name)
    if (files[0]) formData.append('avatar', files[0].file)

    await fetch('/api/profile', { method: 'POST', body: formData })
  }

  return (
    <form onSubmit={handleSubmit}>
      <input value={name} onChange={e => setName(e.target.value)} placeholder="Name" />

      // File drop zone as part of the form
      <div {...getRootProps()}>
        <input {...getInputProps()} ref={inputRef} />
        <p>Drop avatar image here</p>
      </div>

      <button type="submit">Save Profile</button>
    </form>
  )
}
AvatarUpload.tsx 
import { useFileUpload } from 'react-dropkit'

// Single file — adding a new one replaces the previous.
export function AvatarUpload() {
  const { files, getRootProps, getInputProps, inputRef } = useFileUpload({
    multiple: false,
    validation: { accept: ['image/*'], maxSize: 2 * 1024 * 1024 },
  })

  const preview = files[0]?.preview

  return (
    <div {...getRootProps()} style={{ width: 96, height: 96, borderRadius: '50%' }}>
      <input {...getInputProps()} ref={inputRef} />
      {preview
        ? <img src={preview} style={{ width: '100%', borderRadius: '50%' }} />
        : <span>Click to upload</span>
      }
    </div>
  )
}
UploadWithProgress.tsx 
import { Dropzone } from 'react-dropkit'

// Real progress via XHR — works with any backend that supports it.
export function UploadWithProgress() {
  return (
    <Dropzone
      onUpload={(file) => new Promise((resolve, reject) => {
        const xhr = new XMLHttpRequest()
        const formData = new FormData()
        formData.append('file', file.file)

        xhr.upload.onprogress = (e) => {
          if (e.lengthComputable) {
            const pct = Math.round((e.loaded / e.total) * 100)
            // update your own progress state here
          }
        }

        xhr.onload = () =>
          xhr.status === 200 ? resolve() : reject(new Error('Failed'))
        xhr.onerror = () => reject(new Error('Network error'))

        xhr.open('POST', '/api/upload')
        xhr.send(formData)
      })}
    />
  )
}

Every prop,

<Dropzone /> props
PropTypeDescription
validationValidationOptionsFile type, size, and count rules. See ValidationOptions below.
multiplebooleanAllow multiple files. Default: true. Set to false for single-file mode — new file replaces previous.
disabledbooleanDisables the drop zone entirely. Default: false.
onFilesAdded(files: UploadedFile[]) => voidFires when new files pass validation and are added to the list.
onFilesChange(files: UploadedFile[]) => voidFires on every change — add, remove, clear. Always reflects the current files array. Use this to sync external state.
onFileRemoved(id: string) => voidFires when a file is removed from the list.
onUpload(file: UploadedFile) => Promise<void>Your upload handler. The library manages status and progress around your Promise.
onError(error: string, file?: UploadedFile) => voidFires on validation failure or upload error.
classNamestringCustom class applied to the outer wrapper element.
childrenReactNodeOverride the default drop zone UI while keeping all logic.
ValidationOptions
OptionTypeDescription
acceptstring[]Accepted file types. Supports wildcards (image/*), exact MIME types (image/png), and extensions (.pdf). Can mix all three.
maxSizenumberMaximum file size in bytes. e.g. 5 * 1024 * 1024 for 5MB.
minSizenumberMinimum file size in bytes.
maxFilesnumberMaximum total number of files allowed in the drop zone.
useFileUpload() returns
ValueTypeDescription
filesUploadedFile[]The current list of files with status, progress, and preview.
isDragActivebooleanTrue when a file is being dragged over the drop zone.
isDragRejectbooleanTrue when dragged files don't match the accepted types.
getRootProps()HTMLAttributesSpread onto your drop zone container. Handles all drag events, click, and keyboard.
getInputProps()InputHTMLAttributesSpread onto a hidden input element for file browser support.
inputRefRefObject<HTMLInputElement>Attach to the input element so clicking opens the file browser.
addFiles(files)(File[]) => voidProgrammatically add files to the list.
removeFile(id)(string) => voidRemove a file by its ID.
clearFiles()() => voidRemove all files and revoke all preview URLs.
uploadFile(id)(string) => Promise<void>Upload a single file by ID using the onUpload handler.
uploadAll()() => Promise<void>Upload all idle or failed files in parallel.
openFileDialog()() => voidProgrammatically open the native file browser.
UploadedFile object
FieldTypeDescription
idstringUnique identifier for this file entry.
fileFileThe native browser File object. Use file.file to get the raw File for FormData.
previewstring | undefinedBlob URL for image files. Use as img src for thumbnail. Undefined for non-images.
progressnumberUpload progress from 0–100.
statusFileStatus'idle' | 'uploading' | 'success' | 'error'
